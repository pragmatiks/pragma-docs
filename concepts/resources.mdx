## Resource Identity

Every resource has a unique identity:

| Field | Description | Example |
|-------|-------------|---------|
| `provider` | The provider that manages this resource | `gcp` |
| `resource` | The type of resource | `storage` |
| `name` | A unique name you assign | `data-lake` |

## Lifecycle States

Resources move through these states:

| State | Description |
|-------|-------------|
| **DRAFT** | Saved but not submitted for processing |
| **PENDING** | Queued for processing |
| **PROCESSING** | Currently being processed |
| **READY** | Successfully processed |
| **FAILED** | Processing failed |

## Configuration and Outputs

Resources have configuration (what you define) and outputs (what providers produce):

```yaml
# You define this
provider: gcp
resource: storage
name: data-lake
config:
  location: EU
  storage_class: STANDARD

# Provider produces outputs after processing
# outputs:
#   bucket_url: "..."
#   created_at: "..."
```

## Applying Resources

By default, resources are created in DRAFT state:

```bash
pragma resources apply warehouse.yaml
```

To queue for immediate processing:

```bash
pragma resources apply --pending warehouse.yaml
```
